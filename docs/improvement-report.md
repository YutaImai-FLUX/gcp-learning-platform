## 2. ダンジョンシステムの課題

### 良い点（維持すべき）

- **ビジュアル**: マップ、バトルUI、テーマカラー、アニメーション全て最高級（9.5/10）
- **テーマ体系**: 8資格×8テーマ（森/テック/城/火山/氷/海/洞窟/空）が一貫して美しい
- **マップ可視化**: SVGパスアニメーション、ジグザグ配置、パルスアニメーションが秀逸
- **バトルUI**: ダメージポップアップ、HP変色、シェイクアニメーションが気持ちいい

### 問題点

| 問題 | 詳細 | 影響 |
|---|---|---|
| **メカニクスが一次元的** | 正解=勝利、不正解=敗北。戦術・リスクリワードなし | ゲームとしての深みがない |
| **リプレイ性ゼロ** | クリア後に再訪する動機がない | 一度遊んだら終わり |
| **Lab/Demoルーム未実装** | 型定義にあるがauto-clearされるだけ | ルーム種類の多様性が活かされていない |
| **宝箱ルームが空** | XPのみ（アイテムシステムなし） | 期待外れ感 |
| **ボス個性なし** | 通常敵と同じ（HPが150なだけ） | ボス戦の特別感がない |
| **サウンドゼロ** | 完全無音 | ゲーム体験の満足度が大幅に下がる |
| **NPC対話が2行のみ** | キャラクター性・ストーリー性なし | 没入感が薄い |
| **難易度固定** | Normal一択、Hard/Expertなし | やりこみ要素なし |
| **ランク/グレードなし** | 勝利/敗北の二値のみ | 達成感の粒度が粗い |

## 7. まとめ

**このプラットフォームは「UIは完成、コンテンツは未完成」の状態。**

UI/UXとフロントエンド設計は商用レベルの品質だが、学習プラットフォームとしての実用性はコンテンツ量（クイズ87問/フラッシュカード47枚）が致命的に不足している。また、SM-2間隔反復や適応型難易度など高度な学習アルゴリズムが実装済みなのに未接続という「もったいない」状態が複数ある。

**最優先アクション: クイズ問題を300問以上に拡充し、PCA/PDEのモジュールを追加すること。**
これにより学習プラットフォームとしての最低限の実用性が確保され、ダンジョンの戦闘バリエーションも自動的に豊かになる。
